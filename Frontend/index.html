ABOUT HASHMAP
WHY HASHMAP?
Standard Database queries are too slow for high-speed redirection. A HashMap provides O(1) Constant Time access.

  • PUT (Set): Adds links to RAM Cache.
  • GET (Hit): Instantly fetches without touching DB.
  • DELETE (Eviction): Removes "Least Recently Used" (LRU) item.
  • Reverse Lookup: Checks if a URL already exists.

ABOUT MIN HEAP
WHY MIN HEAP?
Efficiently tracks which link expires next. Root contains the earliest-expiring link.
OPERATIONS:
  • Insert (O(log n)): Adds TTL links to queue.
  • Extract-Min (O(log n)): Removes expired root.
